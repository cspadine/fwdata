This is an prototype version of a NoSQL database and web interface for storing, searching, and exporting linguistic fieldwork data. The goal of this project is to create an extremely user-friendly system with a low bar to entry that allows lingusitic reaserchers to benefit from the utility and versatility of database management systems. It is designed to address common issues wth data storage in fieldwork. Currently, the available options for storing fieldwork data falls generally into two categories: specialized software for lingustic data, which tends to require a large amount of set-up and has a steep learning curve, and more general solutions like shared spreadsheets, which provide very little support for data entry, searching, and maintenence needs specific to linguistic data. This application is intended to bridge this gap.

The database itself is built on MongoDB, using Mongoose for schema validation. The application uses Node.js and Express.js.

Current functionality

  • Storing data

    Data is stored in a mongoDB database, which currently supports the following fields:

    • text: a sentence in the target language.
    • morphological breakdown: an array of morphemes that make up the text.
    • interlinear gloss: the corresponding translation or label for each morpheme.
    • free translation: the meaning of the sentence in the medium language.
    • context: any background information given to a consultant while eliciting a sentence.
    • notes: comments from the user about the example.
    • tags: a list of short, searchable phrases relevant to the example.
    • source: the document that the example was taken from, or alternatively, an identifier for the language consultant.
    • reference: an identifier for the particular example.
    • language: the iso code for the target langauge.

  • Searching database

    The data can be searched using criteria from any of the fields above, including multiple criteria per field.

  • Upload data in JSON format

    JSON files of data in compatable format can be directly uploaded into the database.

  • Creating data manually

    Data can be manually entered into the database, in an interface that partially automates the data entry process.

    On entering data, the gloss will be autopopulated based on suggestions from existing data.

  • Deleting data

    Data can be deleted perminantly from the database.

  • Exporting data as JSON file.

Functionality in development

  • Exporting data as text, HTML, LaTeX

    Selected data will be downloadable in mutliple formats so that it can be easily added to documents.

  • Tools for data maintenence

    Users will be able to view all unique lexical items, merge lexical items to regularize spelling and annotation conventions, and standardize glossing according to Leipzig glossing conventions.

  • Built-in citation linking

    Citations for data collected from external sources will be accessable and downloadable along with the data itself.

Next steps

  • Data sharing

    Users will be able to selectively allow other users to access their data.

  • Scraping data from PDFs

    Upload pdfs to add the data they contain and their citation to a database.

  • Support for a broader range of language features

    Expand support for non-standard characters and for including audio and video files.
